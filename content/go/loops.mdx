## The Problem with Manual Work

In Chapter 4, you wrote code to check if *one* student passed. But Riverside Academy has 30 students. Writing 30 separate `if` statements is madness — and what if the class grows to 50?

Loops let you write the logic **once** and run it automatically for every student. This is one of the most powerful ideas in programming.

---

## Go Has One Loop — `for`

Most languages have multiple loop types (`for`, `while`, `do-while`). Go simplifies this: there's only `for`, but it can be written in several styles.

### Classic for — counter-based

```gorun
package main

import "fmt"

func main() {
    // Print seat numbers 1 through 5
    for i := 1; i <= 5; i++ {
        fmt.Println(i)
    }
}
```

Three parts, separated by semicolons:
1. `i := 1` — start condition (runs once before the loop)
2. `i <= 5` — keep looping while this is true
3. `i++` — run after each iteration (shorthand for `i = i + 1`)

## Looping Through Data

Combine what you know: a loop with the array from Chapter 6 (coming up) or scores stored inline:

```gorun
package main

import "fmt"

func main() {
    scores := [5]int{72, 88, 65, 91, 54}

    total := 0
    for i := 0; i < len(scores); i++ {
        total += scores[i]
    }

    average := total / len(scores)
    fmt.Println("Class average:", average)
}
```

## While-Style Loop

Omit the init and post parts and `for` acts like a `while` loop — it keeps going as long as the condition is true:

```gorun
package main

import "fmt"

func main() {
    waitingList := 12
    weeks := 0

    // Admit 3 students per week until the list is empty
    for waitingList > 0 {
        waitingList -= 3
        weeks++
    }

    fmt.Println("Weeks to clear list:", weeks)
}
```

## break — Exit the Loop Early

`break` stops the loop immediately and jumps to the code after it:

```gorun
package main

import "fmt"

func main() {
    scores := [6]int{72, 88, 100, 65, 91, 54}

    for i := 0; i < len(scores); i++ {
        if scores[i] == 100 {
            fmt.Println("Perfect score at position", i)
            break  // No need to keep looking
        }
    }
}
```

---

## continue — Skip and Keep Going

`continue` skips the rest of the current iteration and moves to the next one. Useful for ignoring special cases:

```gorun
package main

import "fmt"

func main() {
    scores := [6]int{72, -1, 88, 65, -1, 91}

    for i := 0; i < len(scores); i++ {
        if scores[i] == -1 {
            continue  // Student was absent — skip
        }
        fmt.Println(scores[i])
    }
}
```

## Combining Loops with if — Counting

This is where chapters 4 and 5 really connect. Use a loop to check every score, and an `if` to count only the ones that pass:

```gorun
package main

import "fmt"

func main() {
    scores := [5]int{72, 88, 65, 91, 54}
    passes := 0

    for i := 0; i < len(scores); i++ {
        if scores[i] >= 60 {  // Rule from Chapter 4
            passes++
        }
    }

    fmt.Printf("%d out of 5 students passed\n", passes)
}
```
