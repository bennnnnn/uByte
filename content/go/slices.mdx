## The Problem Arrays Can't Solve

In Chapter 6, you stored 5 students in a `[5]string` array. It worked perfectly — until a 6th student enrolled. You can't add to a `[5]string`. The size is permanent.

**What if your school keeps growing?**

```go
// This is wrong — you can't grow a fixed-size array
var seats [5]string
seats[5] = "Frank"  // PANIC: index out of range
```

Slices solve this. A slice is like an array, but it can grow and shrink at any time.

---

## Creating a Slice

The difference between an array and a slice is subtle but important: **no number inside the brackets**:

```go
// Array — fixed at 5 forever
var seats [5]string

// Slice — starts empty, can grow
var students []string
```

The most common way to create a slice with initial values:

```gorun
package main

import "fmt"

func main() {
    students := []string{"Alice", "Bob", "Carol"}
    fmt.Println(students)
    fmt.Println("Count:", len(students))
}
```

---

## Adding Students — append

`append` creates a **new, larger slice** with the element added, then returns it. Always assign the result back:

```gorun
package main

import "fmt"

func main() {
    students := []string{"Alice", "Bob", "Carol"}
    fmt.Println("Before:", students)

    students = append(students, "David")
    fmt.Println("After:", students)
    fmt.Println("Count:", len(students))
}
```

You can append multiple values at once:

```go
students = append(students, "Eve", "Frank")
```

## Accessing Elements — Same as Arrays

Everything you learned about array indexing in Chapter 6 works the same way with slices:

```gorun
package main

import "fmt"

func main() {
    students := []string{"Alice", "Bob", "Carol", "David"}

    fmt.Println(students[0])               // First: Alice
    fmt.Println(students[len(students)-1]) // Last: David
}
```

---

## Looping with range

The `range` keyword is the idiomatic way to loop over a slice — cleaner than managing an index manually:

```gorun
package main

import "fmt"

func main() {
    students := []string{"Alice", "Bob", "Carol", "David"}

    for i, name := range students {
        fmt.Printf("Seat %d: %s\n", i, name)
    }
}
```

If you don't need the index, discard it with `_`:

```go
for _, name := range students {
    fmt.Println(name)
}
```

## Removing an Element

To remove an element at index `i`, join the slice *before* it with the slice *after* it:

```gorun
package main

import "fmt"

func main() {
    students := []string{"Alice", "Bob", "Carol", "David"}

    // Remove Bob (index 1)
    students = append(students[:1], students[2:]...)

    fmt.Println(students) // [Alice Carol David]
}
```

- `students[:1]` — everything before index 1 (just Alice)
- `students[2:]` — everything from index 2 onward (Carol, David)
- `...` — spreads the second slice into individual arguments for append

## Slices of Slices — 2D Data

Just like you can have a slice of strings, you can have a slice of slices — great for representing multiple classes:

```gorun
package main

import "fmt"

func main() {
    classes := [][]int{
        {72, 88, 65},  // Class A scores
        {91, 54, 77},  // Class B scores
    }

    for classNum, scores := range classes {
        total := 0
        for _, s := range scores {
            total += s
        }
        fmt.Printf("Class %d average: %d\n", classNum+1, total/len(scores))
    }
}
```

## Arrays vs Slices — When to Use Which

| | Array | Slice |
|---|-------|-------|
| Size | Fixed forever | Can grow and shrink |
| Syntax | `[5]string` | `[]string` |
| Use when | Size is always the same | Size might change |
| Common? | Rare | Very common |

In practice, you'll use slices **most of the time**. Arrays are used when you know the exact count and performance matters at the lowest level.
