## Code That Decides

Until now, your programs have run the same way every time — straight from top to bottom. But real programs need to **make decisions**. Should a student pass or fail? What letter grade did they earn? Is this student eligible for a scholarship?

That's what control flow is for — it lets your program choose different paths based on conditions.

---

## if — Only Run This When True

```gorun
package main

import "fmt"

func main() {
    score := 83

    if score >= 60 {
        fmt.Println("Pass")
    }
}
```

The block inside `{}` only runs when the condition is `true`. If `score` were `50`, nothing would be printed.

---

## if-else — Handle Both Cases

```gorun
package main

import "fmt"

func main() {
    score := 54

    if score >= 60 {
        fmt.Println("Pass")
    } else {
        fmt.Println("Fail")
    }
}
```

One path runs when the condition is true, the other when it's false. The program always takes exactly one path.

---

## else if — Multiple Conditions in Sequence

Riverside Academy awards letter grades. You need to check several score ranges in order — that's what `else if` is for:

```gorun
package main

import "fmt"

func main() {
    score := 83

    if score >= 90 {
        fmt.Println("A")
    } else if score >= 80 {
        fmt.Println("B")
    } else if score >= 70 {
        fmt.Println("C")
    } else if score >= 60 {
        fmt.Println("D")
    } else {
        fmt.Println("F")
    }
}
```

Go evaluates from top to bottom and takes the first condition that's true. Since `83 >= 80` is true, it prints `"B"` and skips everything else.

## switch — Cleaner Than Long if-else Chains

When you're comparing the same variable against many specific values, `switch` is much cleaner than a chain of `else if`:

```gorun
package main

import "fmt"

func main() {
    subject := "SCI"

    switch subject {
    case "MAT", "SCI":
        fmt.Println("STEM")
    case "ENG", "HIS":
        fmt.Println("Humanities")
    default:
        fmt.Println("General")
    }
}
```

One case can match multiple values (separated by commas). The `default` runs if nothing else matches — like the final `else`.

## Combining Conditions: && and ||

Real decisions often involve multiple conditions:

- `&&` — AND: **both** conditions must be true
- `||` — OR: **at least one** condition must be true

```gorun
package main

import "fmt"

func main() {
    score := 92
    gpa   := 3.8

    // Honor roll: high score AND high GPA
    if score >= 90 && gpa >= 3.5 {
        fmt.Println("Honor Roll")
    }

    // Scholarship: high score OR high GPA
    if score >= 85 || gpa >= 3.5 {
        fmt.Println("Scholarship Eligible")
    }
}
```

## Comparison Operators

| Operator | Meaning |
|----------|---------|
| `==` | equal to |
| `!=` | not equal to |
| `>` | greater than |
| `<` | less than |
| `>=` | greater than or equal to |
| `<=` | less than or equal to |

**Important:** `=` assigns a value. `==` compares two values. They look similar but do completely different things.
