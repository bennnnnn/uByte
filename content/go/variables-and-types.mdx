## Variables Are Labeled Boxes

Imagine a row of boxes on a shelf. Each box has a label (the variable name) and contains something (the value). Before you can use a box, you have to create it and tell Go what kind of thing it will hold вҖ” that's the **type**.

## Declaring Variables

### The explicit way: `var`

```go
var name string = "Alice"
var age int = 17
```

This is crystal clear: the variable is named `name`, it holds a `string`, and the value is `"Alice"`.

### The short way: `:=`

Inside a function, `:=` does the same thing in one step вҖ” Go figures out the type from the value:

```gorun
package main

import "fmt"

func main() {
    name := "Alice"   // Go infers: string
    age  := 17        // Go infers: int
    gpa  := 3.8       // Go infers: float64

    fmt.Println(name)
    fmt.Println(age)
    fmt.Println(gpa)
}
```

`:=` is by far the most common form вҖ” you'll use it constantly throughout this course.

## The Four Basic Types

| Type | What it holds | Example |
|------|--------------|---------|
| `string` | text | `"Alice"` |
| `int` | whole numbers | `17`, `0`, `-5` |
| `float64` | decimal numbers | `3.8`, `9.99` |
| `bool` | true or false | `true`, `false` |

```gorun
package main

import "fmt"

func main() {
    name     := "Alice"   // string
    age      := 17        // int
    gpa      := 3.8       // float64
    enrolled := true      // bool

    fmt.Println(name, age, gpa, enrolled)
}
```

## Zero Values вҖ” Go Never Leaves a Box Empty

What happens if you create a variable but don't put anything in it? Go fills it with a safe **zero value** automatically:

| Type | Zero value |
|------|-----------|
| `int` | `0` |
| `float64` | `0` |
| `string` | `""` (empty string) |
| `bool` | `false` |

```gorun
package main

import "fmt"

func main() {
    var score int
    var comment string
    var active bool

    fmt.Println(score)    // 0
    fmt.Println(comment)  // (blank line)
    fmt.Println(active)   // false
}
```

This matters вҖ” if you forget to set a value, you get a predictable result, not a crash.

## Updating Variables

Variables can change. That's the whole point of a *variable*:

```gorun
package main

import "fmt"

func main() {
    score := 78
    fmt.Println(score)  // 78

    score = score + 5   // bonus points
    fmt.Println(score)  // 83

    score += 2          // shorthand for score = score + 2
    fmt.Println(score)  // 85
}
```

Note: updating uses `=` (single equals), not `:=`. The `:=` is only for the *first time* a variable is created.

## Type Conversion вҖ” Explicit, Always

Go **never converts types automatically**. If you need a `float64` as an `int`, you must say so explicitly:

```gorun
package main

import "fmt"

func main() {
    gpa := 3.8
    intGpa := int(gpa)  // decimal part is truncated, not rounded
    fmt.Println(intGpa) // 3
}
```

---

## Constants вҖ” Values That Must Never Change

Some values should be fixed for the lifetime of the program: the minimum passing score, the school name, the maximum GPA. Use `const` for these:

```go
const minPassScore = 60
const schoolName = "Riverside Academy"
```

If you try to change a `const` later, Go will refuse to compile. That's a feature вҖ” it protects you from accidents.
