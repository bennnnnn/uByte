## You just hit a problem

In Chapter 1, you printed your name like this:

```gorun
package main

import "fmt"

func main() {
    fmt.Println("Alex")
}
```

That works fine. But what if you needed to use your name in 10 different places in your program? You'd have to type "Alex" 10 separate times. And the moment you want to change it, you'd have to hunt down all 10 places and update each one.

There has to be a better way. And there is. It's called a variable.

---

## What is a variable?

Think about the contacts app on your phone. You don't memorize your mom's phone number — you store it under the name "Mom". Whenever you want to call her, you just look up "Mom" and the phone fills in the number for you.

A variable is exactly that. It's a name you give to a piece of information so you can use it later without remembering the actual value. You give it a name, you put a value in it, and from that point on you can use the name anywhere you need the value.

In Go, creating a variable looks like this:

```go
name := "Alex"
```

That's it. Three things happening here:

- name is what you're calling it (you pick this)
- := is Go's way of saying "create this variable and put this value in it"
- "Alex" is the value going into it

From this point on, anywhere you write name in your program, Go replaces it with "Alex".

---

## Printing a variable

Remember fmt.Println from Chapter 1? It works exactly the same way with variables. Instead of putting the text directly inside the parentheses, you put the variable name:

```gorun
package main

import "fmt"

func main() {
    name := "Alex"
    fmt.Println(name)
}
```

Run that. You'll see Alex printed. Not the word "name" — the value stored inside it. That's the whole point.

## The four types of data

Every variable in Go holds a specific type of data. Go needs to know what kind of information is going in so it can handle it properly. There are four types you'll use constantly:

---

### Strings — text

A string is any text: names, sentences, email addresses, usernames. You wrap it in double quotes.

```gorun
package main

import "fmt"

func main() {
    firstName := "Alex"
    city := "New York"
    fmt.Println(firstName)
    fmt.Println(city)
}
```

The name "string" comes from the idea of characters strung together in a line. Every character in "Alex" is one bead on a string: A, l, e, x.

---

### Integers — whole numbers

An int holds whole numbers. Ages, scores, counts, years. No decimal point — just a plain number.

```gorun
package main

import "fmt"

func main() {
    age := 24
    score := 87
    year := 2025
    fmt.Println(age)
    fmt.Println(score)
    fmt.Println(year)
}
```

Notice there are no quotes around numbers. Quotes mean text. 24 is a number. "24" is the characters 2 and 4 as text — completely different things in Go's eyes.

---

### Float64 — numbers with decimals

When you need a decimal point — prices, temperatures, GPAs, percentages — you use float64.

```gorun
package main

import "fmt"

func main() {
    price := 9.99
    temperature := 98.6
    gpa := 3.7
    fmt.Println(price)
    fmt.Println(temperature)
    fmt.Println(gpa)
}
```

Go will figure out it's a float64 automatically whenever you write a number with a decimal point. You don't have to say float64 yourself.

---

### Booleans — yes or no

A bool can only ever be one of two values: true or false. That's it. No maybe. No sometimes. Just true or false.

This sounds too simple to be useful, but booleans end up being everywhere. Is this user logged in? Is the payment complete? Does the student have an active enrollment?

```gorun
package main

import "fmt"

func main() {
    isLoggedIn := true
    paymentDone := false
    fmt.Println(isLoggedIn)
    fmt.Println(paymentDone)
}
```

## Updating a variable

The word "variable" literally means something that can vary — that can change. So yes, you can change the value inside a variable after you create it.

The key difference:

- := creates a new variable (use this the first time)
- = updates an existing one (use this every time after)

```gorun
package main

import "fmt"

func main() {
    score := 70
    fmt.Println(score)

    score = 85
    fmt.Println(score)

    score = score + 10
    fmt.Println(score)
}
```

Run that. You'll see 70, then 85, then 95. The same variable, updated three times.

A common mistake is using := when you mean =. If you do score := 85 a second time, Go will complain that you're trying to create a variable that already exists. Just use = for updates.

---

## Using multiple variables together

Programs almost always work with several pieces of information at the same time. You can create as many variables as you need and use them all together:

```gorun
package main

import "fmt"

func main() {
    name := "Alex"
    age := 24
    score := 87.5
    passed := true

    fmt.Println(name)
    fmt.Println(age)
    fmt.Println(score)
    fmt.Println(passed)
}
```

Each variable has its own name, its own type, and its own value — completely independent from the others.
