## Concepts in this chapter

You will learn and practice:

- **select** — receive from whichever channel is ready
- **default case** — non-blocking select / polling
- **Timeout pattern** — `time.After` for cancellable waits
- **Done channel** — signal goroutines to stop
- **Fan-in** — merge multiple channels into one
- **Send cases** — select arms that send, not just receive
- **Nil channels** — disable a select case at runtime
- **Event loop** — `for` + `select` for continuous event processing
- **Context** — `ctx.Done()` as a select case for cancellation

Each section below has a short explanation; then you'll code the same ideas in the exercises (one concept per exercise).

---

## Selecting Between Channels

The `select` statement is like a `switch` for channels. It blocks until one of its cases can proceed, then executes that case. If multiple are ready, it picks one at random.

```go
select {
case v := <-ch1:
    fmt.Println("received from ch1:", v)
case v := <-ch2:
    fmt.Println("received from ch2:", v)
}
```

## The Default Case

Adding `default` makes `select` non-blocking. If no channel is ready, `default` runs immediately — useful for polling without blocking.

## Timeout Pattern

`time.After(d)` returns a channel that receives a value after duration `d`. Combined with `select`, this implements cancellable waits without a timer struct.

## Done Channel Pattern

A `done` channel signals goroutines to stop. When the parent closes or sends to `done`, all goroutines watching it with `select` can exit cleanly.

## Fan-In Pattern

Fan-in merges multiple input channels into a single output channel. Each input gets its own goroutine that forwards values. `select` picks whichever is ready.

## Select with Send Cases

`select` works symmetrically: case arms can send *or* receive. If a send blocks because the receiver is not ready, the other cases are tried.

```gorun
// select-send.go
package main

import "fmt"

func main() {
	ch1 := make(chan int, 1)
	ch2 := make(chan int, 1)

	// Only one channel can receive a value in one select arm
	select {
	case ch1 <- 1:
		fmt.Println("sent to ch1")
	case ch2 <- 2:
		fmt.Println("sent to ch2")
	}
}
```

## Nil Channels in Select

A nil channel blocks forever. You can exploit this to *disable* a case inside `select` at runtime — set the channel variable to `nil` and that case is permanently skipped without removing the code path.

```gorun
// nil-channel.go
package main

import "fmt"

func main() {
	a := make(chan string, 1)
	var b chan string // nil — this case will never fire

	a <- "hello"

	select {
	case v := <-a:
		fmt.Println("a:", v)
	case v := <-b:
		fmt.Println("b:", v) // never reached
	}
}
```

## Select Loop (Event Loop Pattern)

Wrapping `select` in a `for` loop creates an event loop: the goroutine processes one event per iteration and blocks until the next one arrives.

```gorun
// event-loop.go
package main

import (
	"fmt"
	"time"
)

func main() {
	tick   := time.NewTicker(50 * time.Millisecond)
	done   := make(chan struct{})

	go func() {
		time.Sleep(180 * time.Millisecond)
		close(done)
	}()

	count := 0
	for {
		select {
		case <-tick.C:
			count++
			fmt.Println("tick", count)
		case <-done:
			tick.Stop()
			fmt.Println("done after", count, "ticks")
			return
		}
	}
}
```

## Select with Context

The `context` package integrates with `select` through `ctx.Done()`, a channel closed when the context is cancelled or its deadline expires. This is the idiomatic way to propagate cancellation.

```gorun
// context-select.go
package main

import (
	"context"
	"fmt"
	"time"
)

func work(ctx context.Context, result chan<- string) {
	select {
	case <-time.After(200 * time.Millisecond):
		result <- "done"
	case <-ctx.Done():
		result <- "cancelled: " + ctx.Err().Error()
	}
}

func main() {
	ctx, cancel := context.WithTimeout(context.Background(), 100*time.Millisecond)
	defer cancel()

	result := make(chan string, 1)
	go work(ctx, result)
	fmt.Println(<-result) // "cancelled: context deadline exceeded"
}
```
